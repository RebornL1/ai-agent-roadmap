# 每日达标标准

这个仓库的目标不是“每天做一点”，而是每天都要形成可验证的学习进步。

以后每天结束前，必须通过下面 7 道验收门。

## 0. 零基础文件走读完成

每天开始主线项目前，Codex 必须先用新前端学习者能看懂的方式讲清楚项目关键文件。

必须包含：

- 关键文件路径。
- 文件后缀对应什么语言或技术。
- 这个语言或技术是什么。
- 它是为了解决什么问题。
- 官方学习链接。
- 回到项目维度解释：这个文件在当前项目里负责什么。

最低要求：

- 如果是前端项目，至少覆盖 `package.json`、`*.tsx`、`*.css`、配置文件。
- 必须产出或更新项目级 `LEARNING_MAP.md`，用表格说明文件、后缀、技术、问题、链接和项目内作用。
- 如果新增了新技术，必须解释它解决的问题，而不是只写技术名。
- 如果用户没有学过前端，不能默认用户知道 JSX、组件、路由、构建、CSS、状态这些概念。

不达标例子：

- 只说“这里用了 Next.js、React、TypeScript、Tailwind”。
- 只贴代码，不解释后缀、语言和技术用途。
- 只从工程角度讲，不说明它在当前项目里解决什么问题。

## 1. 今日目标明确

必须满足：

- `today.md` 写清今天推进哪个主线项目。
- 今日目标不能太散，只允许 1 个主线项目作为重点。
- 今日目标必须包含“学习理解”和“项目产出”两类结果。

不达标例子：

- 只写“继续学习 AI Agent”。
- 同时做 RAG、MCP、前端、CUDA，目标太散。

## 2. 项目产出可检查

必须满足至少一项：

- 新增或改进代码。
- 新增或改进 README / 设计文档。
- 新增训练题卡、学习日志或面试讲法。

如果当天有代码改动，必须运行对应检查：

- 前端项目：`pnpm lint` 和 `pnpm build`
- 后端项目：对应测试或启动检查
- 文档日：至少检查目录结构和关键文件存在

## 3. 口述训练通过

每天必须至少完成一轮口述训练。

训练标准：

- 每天至少 5 道核心题。
- 每题 3 分：
  - 1 分：方向正确
  - 1 分：结合你的 DBA / 运维背景
  - 1 分：能连接到 AI Agent / 面试表达
- 今日过关线：总分至少 12 / 15。
- 任意低于 2 分的问题，必须重答。

这部分由 Codex 负责提问、评分、纠偏和记录。

## 4. 面试可讲点沉淀

每天必须沉淀至少 3 条“面试可讲点”。

格式：

```text
- 我做了什么
- 为什么这么设计
- 这能证明我什么能力
```

示例：

```text
- 我把 DBA 故障处理流程拆成会话、对话、执行轨迹和工具调用。
- 这样设计是为了让 Agent 的分析过程可见、可审计、可人工介入。
- 这能证明我理解 AI Agent 产品不只是聊天，还包括状态、工具、安全和审计。
```

## 5. 学习日志完成

每天必须在 `logs/` 下生成当天日志。

日志必须包含：

1. 今日完成
2. 今日训练评分
3. 没过的问题
4. 明天计划
5. 面试可讲点
6. Git commit message 建议

## 6. Git 和远程同步

每天结束前必须：

- 本地 git 工作区干净。
- 至少有一个当天 commit。
- 推送到 GitHub 远程。

如果因为网络或认证原因不能推送，必须在当天日志里写清楚阻塞原因。

## 每日最终判定

每天只有两种状态：

- `通过`：7 道验收门全部完成。
- `未通过`：任意一项缺失。

Codex 不能只因为“写了代码”就判定当天完成。必须有训练评分和验证记录。
