# AI Agent 高薪转型作品集仓库

这是一个 6 个月学习和作品集仓库，目标是从 DBA / 运维 / 后端背景，逐步转向 AI Agent 前端和 AI 应用工程方向。

## 仓库定位

这个仓库不是零散学习笔记，而是围绕 5 个主线项目推进的作品集工程。

详细项目规划见：[PROJECTS.md](./PROJECTS.md)

每日达标标准见：[STANDARDS.md](./STANDARDS.md)

## 当前重点

Day 1 从 `projects/dba-agent-workbench` 开始。这个项目是一个面向 DBA 故障处理场景的 AI Agent 工作台骨架，目前只使用模拟数据。

## 主线项目

| 项目 | 定位 | 当前状态 |
|---|---|---|
| `projects/dba-agent-workbench` | AI Agent 前端工作台 | Day 1 已完成三栏骨架 |
| `projects/dba-rag-assistant` | 数据库知识库问答 | 已规划，待启动 |
| `projects/mcp-db-toolkit` | 数据库运维 MCP 工具集 | 已规划，待启动 |
| `projects/ai-agent-ui-components` | AI Agent UI 组件库 | 已规划，待启动 |
| `projects/cuda-npu-lab` | C++ / Linux / CUDA / NPU 副线 | 已规划，待启动 |

## 推进原则

1. 先做能展示的 `dba-agent-workbench`。
2. 再做能证明 AI 应用能力的 `dba-rag-assistant`。
3. 然后把 DBA 能力封装成 `mcp-db-toolkit`。
4. 出现重复 UI 后抽象 `ai-agent-ui-components`。
5. `cuda-npu-lab` 作为副线长期补课，不影响主线交付。

## 每日工作方式

1. 先读 `AGENTS.md`。
2. 再读 `today.md`。
3. 根据 `PROJECTS.md` 选择当天最小、最有价值的任务。
4. 按 `STANDARDS.md` 完成训练和验收。
5. 完成开发、验证，并写学习日志。
6. 提交一次 Git commit，并同步到 GitHub。
