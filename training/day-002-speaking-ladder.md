# Day 2 口述训练梯子

## 使用方法

每道题分 4 步：

1. 跟读：照着标准答案读。
2. 填空：补关键字。
3. 半遮挡：只看骨架说。
4. 独立复述：不看稿说 30-60 秒。

只有完成第 4 步，才算该题通过。

## Q1：这个项目是什么？

### 跟读

> `dba-agent-workbench` 是一个面向 DBA 故障处理场景的 AI Agent 前端工作台。它采用三栏布局：左侧是故障会话列表，中间是用户和 Agent 的对话，右侧展示 Agent 执行轨迹和工具调用。当前版本先用 mock 数据验证产品形态，后续会加入流式输出、工具调用、Human Approval 和报告生成。

## Q2：为什么不能只做普通 Chat 页面？

### 跟读

> 普通 Chat 页面只能看到用户提问和 AI 回答。但 DBA 故障处理不只是问答，还需要看到 Agent 的分析过程、工具调用、风险动作和人工审批。所以这个项目做成三栏：左侧是故障会话列表，中间是 Chat 对话，右侧是 Agent 执行轨迹和工具调用面板。这样设计的价值是让 AI Agent 的过程可见、可审计、可人工介入，更适合数据库生产故障这种高风险场景。

```text
左侧是____，
中间是____，
右侧是____和____。
这样设计的价值是让 AI Agent 的过程____、____、____。
```

### 半遮挡

```text
普通 Chat 的问题是……
DBA 故障处理还需要……
三栏分别是……
这样设计的价值是……
```

### 独立复述评分点

- 普通 Chat 只展示问答
- DBA 场景需要过程和风险控制
- 三栏职责明确
- 可见、可审计、可人工介入

## Q3：Agent 执行轨迹有什么价值？

### 跟读

> Agent 执行轨迹的价值，是让用户知道 Agent 正在做什么、调用了什么工具、有没有风险动作。它把 Agent 的过程从黑盒变成可见时间线，方便排查错误、审计工具调用，也能在高风险步骤让人介入。

### 填空

```text

它把 Agent 从____变成____。
这样可以方便____、____，也能让人在____步骤介入。
```

### 半遮挡

```text
Agent 执行轨迹不是装饰，它是……
它能让用户看到……
它对生产场景的价值是……
```

### 独立复述评分点

- 不是黑盒
- 展示 planning / retrieving / tool calling / approval / report
- 可审计
- 可人工介入

## Q4：Human Approval 是什么？

### 跟读

> Human Approval 就是人工审批。AI Agent 在执行高风险工具调用前，先暂停，把要执行的动作、参数、风险和预期结果展示给人，由人确认后再继续。在 DBA 场景里它非常重要，因为数据库生产操作风险高，AI 不能自动执行所有操作。

```text
在 DBA 场景里它很重要，因为数据库生产操作____，AI 不能____。
```

### 半遮挡

```text
Human Approval 本质上是……
它发生在……
它展示……
人确认后……
DBA 场景必须要它，因为……
```

### 独立复述评分点

- 人工审批
- 高风险动作前暂停
- 展示动作、参数、风险、预期结果
- 人确认后继续
- 数据库生产场景风险高

## Q5：下一步 fake streaming 是什么？

### 跟读

> fake streaming 是模拟模型逐字或分段输出。它不接真实模型，但可以先训练流式 UI、消息状态、停止生成、重新生成和错误重试。这样后续接真实 SSE 或 Vercel AI SDK 时，前端状态机已经准备好了。

### 填空

```text
它暂时不接____，但可以训练____、____、____、____和____。
这样后续接真实____或____时，前端____已经准备好了。
```

### 半遮挡

```text
fake streaming 是……
它现在不做……
它先训练……
它为后续……
```

### 独立复述评分点

- 模拟逐字/分段输出
- 不接真实模型
- 训练流式 UI 状态
- 停止、重试、重新生成
- 为 SSE / Vercel AI SDK 做准备
