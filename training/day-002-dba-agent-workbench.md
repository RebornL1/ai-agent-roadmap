# Day 2 训练题卡：dba-agent-workbench

## 今日目标

让你能把 Day 1 做出来的 `dba-agent-workbench` 讲成一个真实的 AI Agent 产品项目，而不是“我做了个页面”。

## 你的标准项目介绍

先背这个版本，再用自己的话说：

> `dba-agent-workbench` 是一个面向 DBA 故障处理场景的 AI Agent 前端工作台。它把一次数据库故障排查拆成三块：左侧是故障会话列表，中间是人与 Agent 的对话，右侧展示 Agent 的执行轨迹和工具调用过程。这个设计的目标是让 AI 的分析过程可见、可审计、可人工介入，而不是只给用户一个黑盒答案。当前版本先用 mock 数据验证产品形态，后续会逐步加入流式输出、工具调用、Human Approval、报告生成、RAG 和 MCP。

## 第一组：项目定位

### Q1. 这个项目是什么？

回答必须包含：

- AI Agent 前端工作台
- DBA 故障处理场景
- 三栏布局
- mock 数据
- 后续会接流式输出、工具调用、Human Approval

### Q2. 为什么用 DBA 场景？

回答必须包含：

- 这是你的真实经验优势
- DBA 排障天然有 checklist、工具、日志、SQL、复盘
- 适合转成 Agent 工作流
- 比普通聊天机器人更有差异化

### Q3. 它和普通 ChatGPT 页面有什么区别？

回答必须包含：

- 普通 Chat 只展示问答
- Agent Workbench 要展示执行过程
- 工具调用可追踪
- 危险动作需要人工审批
- 最终能生成报告和审计记录

## 第二组：产品结构

### Q4. 为什么设计成三栏？

回答必须包含：

- 左侧管理多个故障会话
- 中间承载主要对话
- 右侧展示 Agent 状态、工具调用、审批和报告
- 这样用户能同时看到输入、推理过程和执行结果

### Q5. 右侧 Agent 执行轨迹有什么价值？

回答必须包含：

- 让 Agent 不再是黑盒
- 用户能看到 planning、retrieving、tool calling、approval、report
- 便于排查 Agent 错误
- 便于审计和复盘

### Q6. 工具调用卡片为什么重要？

回答必须包含：

- Agent 真正有价值的是能调用工具
- 工具调用要展示参数、状态、结果
- 生产场景要能审计
- 数据库场景尤其要强调只读和审批

## 第三组：技术实现

### Q7. 当前用了哪些技术？

回答必须包含：

- Next.js
- React
- TypeScript
- Tailwind CSS
- mock 数据

### Q8. 为什么现在不接真实模型？

回答必须包含：

- Day 1 先验证产品结构
- 避免 API key 和成本问题
- 先把 UI 状态设计清楚
- 后续再接 Vercel AI SDK 或其他模型接口

### Q9. 下一步 fake streaming 是什么？

回答必须包含：

- 模拟模型逐字/分段输出
- 训练流式 UI 状态
- 为真实 SSE / streaming API 做准备
- 可以练停止生成、重新生成、错误重试

## 第四组：Agent 概念

### Q10. Agent 执行轨迹一般有哪些状态？

参考答案：

- planning：制定计划
- retrieving：检索知识
- tool calling：调用工具
- waiting approval：等待人工审批
- analyzing：分析工具结果
- finalizing：生成最终回答或报告

### Q11. Human Approval 是什么？

回答必须包含：

- 人工审批
- 高风险工具调用前暂停
- 用户确认后继续
- 数据库生产场景非常重要

### Q12. 为什么生产数据库场景不能让 Agent 直接执行所有操作？

回答必须包含：

- 数据库操作风险高
- 可能影响生产业务
- 需要权限、只读、审批、审计
- AI 输出可能有误，必须人审核

## 第五组：面试表达

### Q13. 你如何把这个项目讲成你的优势？

参考方向：

> 我的优势是 DBA 和现网稳定性背景，所以我不是做一个泛聊天机器人，而是把真实数据库故障处理流程转成 Agent 工作台。这个项目能体现我对故障排查、工具调用、审批、安全和复盘的理解，也能证明我在补齐 AI 前端和 AI 应用工程能力。

### Q14. 面试官问“这不就是个静态页面吗”，你怎么回答？

参考方向：

> 当前 Day 1 是产品骨架验证，确实先用 mock 数据。但它不是最终状态。这个阶段重点是确认 AI Agent 工作台的信息架构，包括会话、对话、执行轨迹、工具调用、审批和报告。下一步会加入 fake streaming，再接真实 streaming API、RAG 和 MCP 工具链。

### Q15. 面试官问“你为什么不直接接模型”，你怎么回答？

参考方向：

> 因为 AI 应用不是先接模型就完事。Agent 产品里，状态设计、工具调用展示、错误处理、审批和审计都很重要。我选择先把 UI 和交互状态搭好，再接模型，这样后续扩展更稳。
